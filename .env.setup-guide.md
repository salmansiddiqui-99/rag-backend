# Environment Configuration Guide for RAG Chatbot Backend

This guide explains how to fill in the `.env` file with your actual credentials and configuration values.

## Quick Start

1. Open `backend/.env` in a text editor
2. Fill in the required placeholders below with your actual values
3. Save the file
4. Keep `.env` private (never commit to Git)

## Required Configuration Variables

### FastAPI Server Configuration
- **API_HOST**: Server hostname (default: `127.0.0.1` for local development)
- **API_PORT**: Server port (default: `8000`)
- **API_DEBUG**: Enable debug mode (true/false, use `false` in production)

### Database Configuration (Neon Serverless Postgres)
1. Create account at https://neon.tech/
2. Create a new project
3. Copy connection string from Neon dashboard
4. Paste into `DATABASE_URL` (format: `postgresql://user:password@host:port/database`)

Example:
```
DATABASE_URL=postgresql://admin:mypassword@ep-xyz.us-east-2.neon.tech:5432/mydb
```

### Vector Database Configuration (Qdrant Cloud)
1. Create account at https://cloud.qdrant.io/
2. Create a new cluster (Free Tier available: 1M vectors)
3. Copy API URL and API Key from dashboard
4. Fill in `QDRANT_URL` and `QDRANT_API_KEY`
5. Set `QDRANT_COLLECTION_NAME` (default: `physical_ai_chapters`)

Example:
```
QDRANT_URL=https://xyz-xyz-xyz.qdrant.io
QDRANT_API_KEY=ey1234567890abcdef...
QDRANT_COLLECTION_NAME=physical_ai_chapters
```

### LLM Configuration - OpenRouter (Required)
1. Create account at https://openrouter.ai/
2. Generate API key from https://openrouter.ai/keys
3. Fill in `OPENROUTER_API_KEY`
4. Set `OPENROUTER_MODEL` (e.g., `mistralai/devstral-2512:free`, `mistralai/mistral-7b-instruct:free`)
5. The `OPENROUTER_BASE_URL` is already set to `https://openrouter.ai/api/v1`

Example:
```
OPENROUTER_API_KEY=sk-or-v1-abc123xyz...
OPENROUTER_MODEL=mistralai/devstral-2512:free
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
```

Note: OpenRouter provides access to multiple models through a unified API, including free tier options.

### CORS Configuration
Configure which domains can access the backend API:
- **Local Development**: `http://localhost:3000,http://127.0.0.1:3000`
- **Production**: `https://salmansiddiqui-99.github.io/hackathon1-Q4`

Comma-separated list:
```
CORS_ORIGINS=http://localhost:3000,https://salmansiddiqui-99.github.io/hackathon1-Q4
```

### RAG Configuration
- **RAG_SIMILARITY_THRESHOLD**: Minimum relevance score (0.0-1.0, default: 0.5)
  - Higher = stricter (fewer hallucinations, may miss valid answers)
  - Lower = more lenient (more answers, risk of hallucination)
- **RAG_TOP_K_CHUNKS**: Number of chunks to retrieve (default: 5)
- **RAG_MAX_RESPONSE_TOKENS**: Maximum response length (default: 1000)

### Chat Session Configuration
- **CHAT_LOG_RETENTION_DAYS**: How long to keep chat logs (default: 90 days for GDPR compliance)
- **CHAT_MAX_MESSAGES_PER_SESSION**: Max messages per conversation (default: 50)

### Rate Limiting
- **RATE_LIMIT_REQUESTS_PER_MINUTE**: Requests per minute per user (default: 10 req/min)

### Feature Flags
Enable/disable features:
- `FEATURE_TEXT_SELECTION_MODE=true` - Allow users to highlight text
- `FEATURE_PERFORMANCE_LOGGING=true` - Log latency metrics
- `FEATURE_HALLUCINATION_DETECTION=true` - Detect hallucinations
- `FEATURE_STREAMING_RESPONSES=true` - Stream responses token-by-token

### Logging Configuration
- **LOG_LEVEL**: DEBUG, INFO, WARNING, ERROR, CRITICAL (default: INFO)
- **LOG_FORMAT**: json or text (default: json)

### Security
- **SECRET_KEY**: Session encryption key
  - Generate with: `openssl rand -hex 32`
  - Keep this secret! Never commit to Git.
- **ENABLE_HTTPS**: Use HTTPS (false for local dev, true for production)

## Step-by-Step Setup

### 1. Create Neon Postgres Database
```bash
# Visit https://neon.tech/
# Sign up and create a project

```

### 2. Create Qdrant Cloud Instance
```bash
# Visit https://cloud.qdrant.io/
# Sign up and create a cluster (Free Tier: 1M vectors)
# Copy API URL and Key
```

### 3. Create OpenRouter API Key
```bash
# Visit https://openrouter.ai/keys
# Create new API key
# Copy and paste into OPENROUTER_API_KEY
```

### 4. Generate Secret Key
```bash
# Run in terminal/powershell:
openssl rand -hex 32

# Paste output into SECRET_KEY field
```

### 5. Update CORS Origins
```
# For local development:
CORS_ORIGINS=http://localhost:3000,http://127.0.0.1:3000

# For production (GitHub Pages):
CORS_ORIGINS=https://salmansiddiqui-99.github.io/hackathon1-Q4
```

## Validation Checklist

Before running the application, verify:
- [ ] DATABASE_URL is a valid Postgres connection string
- [ ] QDRANT_URL and QDRANT_API_KEY are set and accessible
- [ ] OPENROUTER_API_KEY is set and valid
- [ ] OPENROUTER_MODEL is set (e.g., mistralai/devstral-2512:free)
- [ ] CORS_ORIGINS includes your frontend domain
- [ ] LOG_LEVEL is set (default: INFO)
- [ ] SECRET_KEY is generated and unique
- [ ] Feature flags are enabled as needed

## Security Best Practices

1. **Never commit `.env` to Git**
   - Add `backend/.env` to `.gitignore`
   - `.env` is excluded by default in this project

2. **Use different values per environment**
   - Development: Local Neon/Qdrant instances
   - Production: Cloud instances with higher limits

3. **Rotate API keys regularly**
   - Update OPENROUTER_API_KEY quarterly
   - Monitor usage in OpenRouter dashboard

4. **Use strong SECRET_KEY**
   - Generate with `openssl rand -hex 32`
   - Never reuse across projects

## Troubleshooting

### "Connection refused" to Neon
- Verify DATABASE_URL format is correct
- Check network connectivity
- Ensure IP whitelist allows your machine

### "Unauthorized" to Qdrant
- Verify QDRANT_API_KEY is correct
- Check Qdrant Cloud dashboard for API key
- Ensure QDRANT_URL matches your instance

### "Invalid API key" from OpenRouter
- Verify OPENROUTER_API_KEY format (should start with `sk-or-v1-`)
- Check OpenRouter account has valid API key
- Ensure API key has sufficient quota

## Production Deployment

When deploying to production (Render, Railway, Vercel):

1. Set environment variables in cloud provider dashboard
2. Use environment-specific values (prod Neon, prod Qdrant)
3. Set `ENABLE_HTTPS=true`
4. Set `API_DEBUG=false`
5. Use strong, unique `SECRET_KEY`
6. Update `CORS_ORIGINS` to production domain

Example production config:
```
ENABLE_HTTPS=true
API_DEBUG=false
DATABASE_URL=postgresql://prod_user:prod_password@prod.neon.tech:5432/prod_db
QDRANT_URL=https://prod-qdrant.instance.io
LOG_LEVEL=WARNING
CORS_ORIGINS=https://salmansiddiqui-99.github.io/hackathon1-Q4
```
